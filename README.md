# Python Data Type Fundamentals
- typecasting turning one data type into another

## Strings
- textual data, LIST/sequence of characters
- denoted/delineated with Quotes (single, double triple)
- If a string has has quotes, use the opposite type of quotes to denote string to avoid errors.
- space is also a string char
- Python Indexer - [start:stop:step]
- Strings are IMMUTABLE, every time you edit one, you creat a brand new string

## String Methods
- `str.lower()` - lowercases a string
- `str.upper()` - upppercarse
- `str.replace()` - replace chars in a string
- `str.strip()` - removes white space from beginning and end
- `str.split()` - splits a string into substring based on a delimiter (symbol)
- `str()` - typecasting to a string
- `list()` - turn strings into a list on EVERY CHARACTER

## String AND List Methods/Funtion
- `len()` - function gives length of list or string

## Lists
- `[]` - denote a list
- element is a single index of an list
- zero of more elements in an list
- lists always start counting at 0
- MUTABLE objects, which means they are changed in place
- Can put ANY data type/object in a string

# List methods
- `append()` - adding single element to end
- `extend()` - adding multiple elements from an iterable to the end of a list
- `[].insert()` - add item to a specified in in the list
- `[].remove()` - removes items from an array


## Dictionaries
- Keyed Collections of data
- can be of mixed data types

## Primitive Data type
- Only store one type of data, single data type
- passed by value
- String
- Ints/Floats
- None
- Bool
- Char

## Non_Primitive Data-Types / Data Collections
- Collections of different types of data
- passed by reference

- Lists
    - Ordered and indexed by numbers
    - Always in the same ordered, each piece of data is assigned a number and stays in line
- Dictionaries / Objects
    - keyed collection. Key value pairs
- Tuples

# Definitions
- Traverse - to look at a list/dict/string one item at a time

- lists are indexed
- dictionaries are keyed (key : value pair)
